Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

User

Register User

Example request:
curl --request POST \
    "http://localhost/api/user/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"first_name\": \"Patryk\",
    \"last_name\": \"Gajewski\",
    \"phone_number\": \"777878787\",
    \"email\": \"example@gmail.com\",
    \"country\": \"Poland\",
    \"town\": \"Warsaw\",
    \"password\": \"Qwerty123#\",
    \"password_confirmation\": \"Qwerty123#\"
}"

const url = new URL(
    "http://localhost/api/user/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "first_name": "Patryk",
    "last_name": "Gajewski",
    "phone_number": "777878787",
    "email": "example@gmail.com",
    "country": "Poland",
    "town": "Warsaw",
    "password": "Qwerty123#",
    "password_confirmation": "Qwerty123#"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

POST api/user/register

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

first_name   string   

User first name. Must not be greater than 64 characters. Must be at least 2 characters. Example: Patryk

last_name   string   

User last name. Must not be greater than 64 characters. Must be at least 2 characters. Example: Gajewski

phone_number   string   

User phone number. Must be at least 9 characters. Must not be greater than 12 characters. Example: 777878787

email   string   

User email. Must be a valid email address. Example: example@gmail.com

country   string   

User country of origin. Possible to add (poland,germany,spain,portugal). Example: Poland

Must be one of:
  • poland
  • germany
  • spain
  • portugal
town   string  optional  

Town name where user live. Example: Warsaw

password   string  optional  

Password should have min one capital, number and special character. Example: Qwerty123#

password_confirmation   string   

Confirmation password which has to be the same like password above. The value and password must match. Example: Qwerty123#

    --header "Accept: application/json" \
